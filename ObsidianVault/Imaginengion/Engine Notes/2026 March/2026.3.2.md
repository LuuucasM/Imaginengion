- Changed event managers over to an abstract implementation of event manager
	- the reasons is i consider events to be on of the main pillars of the engine but before each event manager had its own unique implementation but since its a main pillar it should be abstracted and then the abstraction can be iterated apon to optimize in the future
- in the ECS I changed a fundamental part of the abstraction. I changed SkipField to be a component that is directly in the ECS (meaning the ECS now has 3 components, parent, child, skipfield)
	- The reason why i made the change is before I just had a SparseSet of SkipFields that was separate from the ECS which seemed to overlap with the built in functionality of component arrays so why not move SkipFields into a component array so that the current usage of skipfields doesnt change but now since the skipfields are a component it can be used in a much larger different kinds of applications
	- a second motivation is I wanted to change the GetGroup functionality to use the skipfields to be able to early cut components who were never going to be a valid part of the overall group to begin with. This means now during the ".Component" branch of GetGroup I can filter the entities obtained from the internal_array by a mask which represents the queries final ideal grouping. This means that the other branches in GetGroup will have less entities to compare potentially saving computation
	- These combined means that SkipFields almost become like a psudo archetype which means that potential ECS features in the future that I want to implement can be implemented flexible depending on the feature if it is faster to implement in a contiguous array system or archetype system maybe (not sure but this is the idea :))
